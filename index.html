DEPLOYING TO GITHUB PAGES (cross-device + Supabase sync)

1
Go to github.com/new and create a new repository — name it anything, e.g. btc-advisor. Keep it public.
2
Download this HTML file. Rename it to index.html. In your new repo, click Add file → Upload files and upload index.html.
3
Go to Settings → Pages. Under "Branch" select main and click Save. GitHub will give you a URL like https://yourusername.github.io/btc-advisor.
4
Open that URL on any device — phone, tablet, computer. Market data loads automatically.
5
For cross-device prediction history: Set up Supabase (free at supabase.com), run the SQL from Settings, then add your Supabase URL + anon key in ⚙ Settings on each device. History syncs automatically.
6
When you get a new build, just re-upload index.html to the same repo. GitHub Pages updates within ~60 seconds.
Note: Your Anthropic API key and Supabase credentials are stored in each browser's localStorage — you'll need to enter them once per device in Settings. They are never sent anywhere except directly to their respective APIs.
